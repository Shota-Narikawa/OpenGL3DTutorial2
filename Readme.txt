//=============================================================================
//　①制作意図
//=============================================================================
    授業課題は動かない敵出現させて、近づいて倒すだけのものでした。
　　ただ単に敵を倒していくだけでなく少しでもプレイヤーが悩んで考えた先に待つ
    クリアの達成感を感じてプレイしてもらいたくてタワーディフェンスにしました。
//=============================================================================
//　②アプリケーションの内容
//=============================================================================
　　敵をすべて倒すとクリアです。　　
　　ただし、敵はステージ上にある地蔵のオブジェクトを破壊する行動をします。
　　オブジェクトはランダム配置で３個存在します。それらの体力はリンクしており、
　　リンクした体力が無くなるとゲームオーバーとなります。
　　オブジェクトの配置を把握し、うまく立ち周り敵を倒す必要があります。
//=============================================================================
//　③開発環境
//=============================================================================
    C++
    Visual Studio 2017 ver15.9.20

//=============================================================================
//　操作方法
//=============================================================================
//　・キーボード操作

　　WASD　―　移動（プレイ中＆メニュー画面のUI操作）
　　UHJK　―　各スキル攻撃
　　Q　　 ―　視点を左回転
　　E　　 ―　視点を右回転
　　L　　 ―　ジャンプ
　　SPACE ―　メニュー画面を開く
　　ENTER ―　決定キー

　　・ゲームパッド操作

　　十字キー　―　移動（プレイ中＆メニュー画面のUI操作）
　　ABXY　　　―　各スキル攻撃
　　LB　　　　―　視点を左回転
　　RB　　　　―　視点を右回転
　　RT　　　　―　ジャンプ
　　BACK　　　―　メニュー画面を開く
　　START　　 ―　決定キー

　　・デバック用ボタン（キーボードのみ）
　　Z　―　クリア条件
　　X　―　防衛ライン体力の減少
　　C　―　HPの減少
　　V　―　MPの減少
　　B　―　レベルアップ

//=============================================================================
//　④起動方法
//=============================================================================
    ・Gitからのダウンロード
　　Clone or doenlowdからDownload　ZIPで保存
　　保存されたOpenGL3DTutorial2.master.zipを解凍し
　　exeフォルダにあるOpenGL3DTutorial2.exeを起動

　　・visual studioで起動させる場合は
　　　ソリューションプラットフォームをx86で起動お願いいたします。
　　
　　・その他はexeフォルダにあるOpenGL3DTutorial2.exeを起動
　　
    ※もしゲームが立ち上がらない時はコンソール画面でEnterキーを押して下さい。

　　タイトルからGAME　STARTを押下後、
　　操作確認ができたら本編を始めるでスタート
//=============================================================================
//　⑤工夫した点
//=============================================================================
　　当初は守るべきオブジェクトは１つだったのですが、広いステージで
　　１つを見つけるとそこで待ち伏せするだけで終わってしまうので、
　　守るべきオブジェクトを３つに変更し、敵がランダムでそれぞれの
　　オブジェクトに向かうような行動にして、ゲーム性の向上を図りました。
　　また、プレイ画面のスプライトでバー状にしてを使ったUIや
　　スキルコマンドの部分など、プレイヤーが直感的に
　　わかりやすいようにUI面の工夫をしました。

　　ステートやシーン移行を駆使して、メニュー画面やステージ変更の
　　部分のシーケンスを上手くまわすことができました。
